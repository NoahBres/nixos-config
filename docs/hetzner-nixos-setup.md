# Hetzner NixOS Setup

## Overview

The Hetzner server (`ubuntu-4gb-hel1-1`, ID: 126013066) runs NixOS, managed via this flake.
Server: CX23, 2 vCPU, 4 GB RAM, Helsinki (hel1). IPv6-only by default (no public IPv4).

## Normal workflow

**Deploy config changes:**
```sh
nixos-rebuild switch --flake .#hetzner --target-host root@hetzner --use-remote-sudo
```

**SSH access:**
```sh
ssh root@hetzner       # via Tailscale (primary)
ssh root@<IPv6>        # direct IPv6 (requires your machine to have IPv6)
```

## Reprovisioning from scratch

Use this when rebuilding the server entirely (e.g. new Hetzner server, corrupted install).

### Prerequisites

- Nix with flakes enabled (Determinate Systems installer)
- Tailscale auth key from tailscale.com/admin/settings/keys (ephemeral + reusable)
- Hetzner API token from 1Password: `Hetzner - Default - API Token`

### The connectivity problem

Your Mac has no native IPv6, so it can't reach the server's bare IPv6 after kexec
drops Tailscale. The fix: temporarily assign a public IPv4 to the server.

### Step-by-step

**1. Get the API token:**
```sh
TOKEN=$(op item get "Hetzner - Default - API Token" --reveal --format json | \
  python3 -c "import sys,json; data=json.load(sys.stdin); \
  print(next(f['value'] for f in data['fields'] if f.get('value') and len(f['value']) > 10))")
```

**2. Power off the server:**
```sh
curl -s -X POST -H "Authorization: Bearer $TOKEN" \
  https://api.hetzner.cloud/v1/servers/126013066/actions/poweroff
```

**3. Create a temporary Primary IPv4 (costs ~€0.50/mo, delete after):**
```sh
curl -s -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"hetzner-temp-ipv4","type":"ipv4","datacenter":"hel1-dc2","assignee_type":"server"}' \
  https://api.hetzner.cloud/v1/primary_ips
# Note the returned `id` and `ip`
```

**4. Assign the IPv4 to the server (use the `id` from step 3):**
```sh
curl -s -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"assignee_id": 126013066, "assignee_type": "server"}' \
  https://api.hetzner.cloud/v1/primary_ips/<id>/actions/assign
```

**5. Power on:**
```sh
curl -s -X POST -H "Authorization: Bearer $TOKEN" \
  https://api.hetzner.cloud/v1/servers/126013066/actions/poweron
```

**6. Wait ~20s for Ubuntu to boot, then add your Mac's SSH key via Tailscale:**
```sh
ssh root@hetzner "echo '<your ~/.ssh/id_rsa.pub>' >> ~/.ssh/authorized_keys"
```

**7. Run nixos-anywhere targeting the IPv4:**
```sh
cd ~/Developer/nixos-config
nix run github:nix-community/nixos-anywhere -- \
  --flake .#hetzner \
  --generate-hardware-config nixos-generate-config ./hosts/hetzner/hardware-configuration.nix \
  --target-host root@<IPv4>
```
nixos-anywhere will kexec into the NixOS installer, reconnect via IPv4 (DHCP), partition
with disko, install, and reboot.

**8. Commit the generated hardware-configuration.nix and push.**

**9. After reboot, authorize Tailscale:**
```sh
ssh root@<IPv4> "tailscale up --auth-key=<key> --ssh --hostname=hetzner"
```

**10. Verify Tailscale works, then delete the temporary IPv4:**
```sh
curl -s -X DELETE -H "Authorization: Bearer $TOKEN" \
  https://api.hetzner.cloud/v1/primary_ips/<id>
```

## Troubleshooting

### Can't SSH after nixos-install — key rejected

NixOS manages `authorized_keys` declaratively. The file is regenerated by the activation
script on every `nixos-rebuild switch` (but NOT on plain reboots). To get emergency access:

**Option A — Enable Hetzner rescue mode (generates a temporary root password):**
```sh
RESCUE=$(curl -s -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"type": "linux64"}' \
  https://api.hetzner.cloud/v1/servers/126013066/actions/enable_rescue)
echo "$RESCUE" | python3 -c "import sys,json; print(json.load(sys.stdin)['root_password'])"

curl -s -X POST -H "Authorization: Bearer $TOKEN" \
  https://api.hetzner.cloud/v1/servers/126013066/actions/reboot
```
Then SSH with `sshpass -p '<password>' ssh root@<IPv4>`.

**From rescue, update the NixOS config then reinstall in-place:**
```sh
# In rescue shell:
source /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh  # if Nix was installed
# Install Nix if not present:
curl -fsSL https://install.determinate.systems/nix | sh -s -- install linux --no-confirm --init none

source /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh
mount /dev/sda2 /mnt
# sda1 is a raw BIOS boot partition — don't try to mount it
nix shell nixpkgs#nixos-install-tools --command nixos-install \
  --flake github:NoahBres/nixos-config#hetzner \
  --no-root-password
reboot
```

### Hetzner SSH key (1Password) format incompatible with macOS

The `Hetzner SSH` key in 1Password is stored as PKCS#8 (`-----BEGIN PRIVATE KEY-----`)
which macOS's LibreSSL-based `ssh-keygen` and `openssl` reject. The Python `cryptography`
library also rejects it (non-standard optional field in the ASN.1 structure).

**Workaround:** Use the Mac's default RSA key (`~/.ssh/id_rsa`) instead. It is listed in
`hosts/hetzner/configuration.nix` under `authorizedKeys`. The Hetzner ed25519 key is also
listed for completeness but is effectively only usable from Linux systems.

**Note:** `op item edit` does not support SSH Key category items — rename them in the
1Password app directly.

### nixos-anywhere kexec loses network (IPv6-only server, Mac without IPv6)

After kexec, the NixOS installer uses DHCP. With no public IPv4 and the Mac having no
native IPv6, the Mac loses the route to the server. Fix: assign a temporary public IPv4
before running nixos-anywhere (see reprovisioning steps above).

### "You cannot have duplicated devices in mirroredBoots"

Caused by setting `boot.loader.grub.device` in `configuration.nix` while disko also
configures it automatically from the `EF02` BIOS boot partition in `disk-config.nix`.
Fix: do not set `boot.loader.grub.device` manually — just set `boot.loader.grub.enable = true`.
